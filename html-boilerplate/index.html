Intro to Programming

What is Programming?

Essentially programming is telling a computer what to do. Programming is the process of creating a set of instructions that tell a computer how to perform a task.
Programming can be done using a variety of computer "languages," such as Python, Rust, Javascript, Java, Swift, C# and C++.

Fundamental Concepts

Programming involves basic concepts that are present in most if not all languages. The include but are not limited to:

  • Variables
  • Data Types
  • Data Structures
  • Operators
  • Expressions
  • Statements
  • Conditionals
  • Loops
  • Functions

Other Concepts

There are other concepts that are in some but not all languages. Whether or not a language has these concepts is usually dependent on the paradigm of the language. These include but are not limited to:

  • Classes
  • Objects
  • Methods
  • Interfaces[1]
  • Enumerations

Paradigms in Programming

So what are these paradigms we keep hearing about?
A paradigm is a way of thinking about something that shapes how we interpret information and make decisions. In the context of programming, a paradigm is the way the programmer thinks about the operations, which shapes how they give the computer instructions and determins how the code is written.
There are many different paradigms in programming. A language can belong to multiple paradigms, indicating multiple ways of writing code in that language. Such languages are termed "multiparadigm". Some of the most common paradigms in programming are:

  • Imperative Programming
  • Declarative Programming
  • Functional Programming
  • Object-Oriented Programming
  • Procedural Programming
  • Event-Driven Programming
  • Logic Programming
  • Aspect-Oriented Programming
  • Concurrent Programming

Translation

When you write code, you are writing in a language that you can read and understand. The computer however understands only machine code, so your code has to be "translated" into machine code that the computer understands. There are two ways that a computer can run code:

  1. compilation
  2. interpretation
Compiled languages are languages that are compiled into machine code (by a compiler), which is platform-specific code that the computer can understand. Interpreted languages are languages that are interpreted by a program (called the interpreter), which is then run by the computer.
Compiled languages (eg C/C++) are generally much faster than interpreted languages,but interpreted languages (eg Python) are generally easier to write and debug but the compiled code will only run on a particular platform as machine code is platform-specific.
Some languages (eg Java) are both compiled and interpreted. These languages are compiled into an intermediate language, which is then interpreted by another program. This allows the code to be run on multiple platforms without having to be recompiled for each platform.

Typing

When you write code, you'll use different data types. Typing is a way for the interpeter to keep track of the types. There are two types of typing in programming:

  1. static typing
  2. dynamic typing
Static typing is when the type of a variable is determined at compile time.
Dynamic typing is when the type of a variable is determined at runtime.
Static typing is generally considered to be safer than dynamic typing, but dynamic typing is generally considered to be easier to write and debug.
There are also languages that are both statically and dynamically typed. These languages are statically typed, but the type of a variable can be inferred at compile time.
There also two different approaches to type checking. These are:
  1. strong typing
  2. weak typing
Strong typing requires that the types of variables and expressions be checked at compile time. This means that the compiler will not allow you to assign a value of one type to a variable of another type. Strong typing can help to prevent errors and make code more robust.
Weak typing does not require that the types of variables and expressions be checked at compile time. This means that the compiler may allow you to assign a value of one type to a variable of another type. Weak typing can make code more flexible, but it can also make it more error-prone.
